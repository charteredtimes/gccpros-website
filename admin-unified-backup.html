Data Security Posture

Reference summary of how the GCC Database protects member and company data.

  • Encryption at rest: Supabase Postgres on AES-256 encrypted disks (platform default).
  • Encryption in transit: Every API call uses TLS 1.2+. No plaintext over the wire.
  • Service-role key: Stored server-side only (Edge Function secrets) — never sent to a browser.
  • Admin key: Required for every admin action in this console. Rotate periodically.
  • Row-level security: Enabled on all tables; end-users read only via credit-checked proxy endpoints.
  • Field-level filtering: Reveal returns only the locked B2C fields; other columns never leave the server.
  • Watermarking + copy friction: Reveal pages show viewer email + timestamp; right-click/selection blocked.
  • Audit trail: Every credit spend, reveal, search, login, and admin action is logged with metadata.
  • Backups: Supabase daily backups (7-day retention); Pro tier adds point-in-time recovery.
  • Rate limits: Search and reveal throttled per user; monthly reveal cap enforced.
  • Payments: Card data is entered on Razorpay’s checkout only — it never touches our servers.
