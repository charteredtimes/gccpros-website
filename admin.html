Data security posture

  • Encryption at rest: All Supabase Postgres data is stored on AES-256 encrypted disks. This is Supabase platform-default; no extra config needed.
  • Encryption in transit: Every API call (browser → Apps Script, Apps Script → Supabase, browser → Razorpay) uses TLS 1.2+. No plaintext over the wire.
  • Service-role key: Stored in Apps Script Script Properties. Never sent to a browser. Bypasses Postgres row-level security. Anyone with this key has full DB access — treat like a root password.
  • Admin key: Required to call any admin endpoint (this panel, approve/reject/suspend, grant credits, user history). Stored as GCP_ADMIN_KEY in Script Properties. Rotate periodically.
  • Row-level security: Enabled on every table. Authenticated end-users cannot read the full companies table directly — only through Apps Script proxy endpoints, which enforce credit checks and audit logging.
  • Field-level filtering: Reveal endpoint returns only the 12 B2C fields locked on May 16. The other 32 columns (revenue, leader emails, audit notes, etc.) are never sent to a non-admin browser session.
  • Watermarking: Every revealed company card shows the viewer's email + timestamp + "Confidential" diagonally across the page. Designed to discourage screenshot leaks.
  • Copy protection: Right-click disabled, text selection blocked, copy event intercepted on the reveal pages. Determined users can still inspect element — this is friction, not encryption.
  • Audit trail: Every credit spend, reveal, search, login, and admin action is logged with timestamp + metadata. Retention is unlimited (no auto-purge configured).
  • Backups: Supabase Free tier provides daily backups (retained 7 days). Pro tier adds point-in-time recovery for the last 7 days at any second.
  • Rate limits: Search is throttled to 60/min and 600/hour per user. Reveal is capped at 50 unique companies per user per calendar month (resets on the 1st).
  • Payment data: Card numbers are entered directly on Razorpay's checkout — never touch your servers. You only see the payment_id (a non-sensitive reference) and the amount.

What I would harden next

  • Rotate the service-role key — Supabase lets you regenerate it. Currently in your Apps Script Script Properties.
  • Add 2FA to your Supabase account — under Supabase → Account Settings.
  • Enable email alerts on Razorpay refunds / chargebacks — caught early avoids credit-stuffing exploits.
  • Periodically export the user_history CSV for compliance evidence; especially if anyone files a DSAR (data subject access request).
